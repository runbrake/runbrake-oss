# RunBrake OSS v0.1.2 Agent Session Receipts

RunBrake OSS v0.1.2 makes local RunBrake checks visible and queryable inside supported OpenClaw and Hermes agent sessions.

## Highlights

- Adds privacy-safe `CheckReceipt` summaries for local policy, install, and runtime-observation responses.
- Shows compact startup, install, runtime, watcher, and fail-open receipt notices in supported OpenClaw and Hermes sessions.
- Confirms the local sidecar, local policy engine, local install/runtime decisions, local receipts, and local signed audit events are free/open-core.
- Ships the OpenClaw `runbrake-security` skill so users can ask their agent for RunBrake status, recent receipts, scans, watcher checks, sidecar setup, enforcement guidance, and reports.
- Expands the Hermes `/runbrake-security` skill and helper for the same operator workflow.
- Adds watcher digest rendering for skills and plugins added outside supported install hooks.
- Keeps receipt visibility local with `RUNBRAKE_RECEIPTS=quiet`, `RUNBRAKE_RECEIPTS=all`, or `RUNBRAKE_RECEIPTS=off`.

## Install

Install with Homebrew:

```bash
brew update
brew install runbrake/tap/runbrake
runbrake doctor --path ~/.openclaw
runbrake doctor --ecosystem hermes --path ~/.hermes
```

The same `runbrake` CLI package supports OpenClaw and Hermes. Hermes does not have a separate Homebrew package; users install the Hermes runtime adapter and `runbrake-security` skill into `~/.hermes` only when they want receipt notices and policy checks inside Hermes sessions.

Or use a checksummed binary release:

```bash
curl -fsSLO https://github.com/runbrake/runbrake-oss/releases/download/v0.1.2/runbrake_v0.1.2_darwin_arm64.tar.gz
curl -fsSLO https://github.com/runbrake/runbrake-oss/releases/download/v0.1.2/runbrake_v0.1.2_checksums.txt
grep " runbrake_v0.1.2_darwin_arm64.tar.gz$" runbrake_v0.1.2_checksums.txt | shasum -a 256 -c -
tar -xzf runbrake_v0.1.2_darwin_arm64.tar.gz
./runbrake doctor --path ~/.openclaw
./runbrake doctor --ecosystem hermes --path ~/.hermes
```

## Limits

- Receipts are local summaries, not a hosted audit ledger.
- Scanner, doctor, registry, watch, and assessment commands do not require a sidecar. Runtime enforcement and in-session receipts are optional adapter behavior and fail open locally when the sidecar is unavailable.
- Hosted dashboards, Slack/web approvals, team inventory, retained audit history, private catalogs, enterprise integrations, and hosted isolation are commercial control-plane scope.
- Static findings and dependency advisories are review signals, not proof that a skill is actively malicious.
- Runtime observations and receipts are metadata-first and intentionally omit raw prompts, raw tool arguments, file contents, package bodies, memory contents, and unredacted secrets.
- Live smoke checks are opt-in and use isolated temporary OpenClaw or Hermes homes.
