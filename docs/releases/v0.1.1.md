# RunBrake OSS v0.1.1 Scanner Evidence Hardening

RunBrake OSS v0.1.1 hardens scanner evidence before broader developer-preview usage.

## Highlights

- Adds metadata-only runtime observation contracts to the OpenClaw policy adapter.
- Posts runtime observations from `before_tool_call` before policy decisions and fails open when the local observation endpoint is unavailable.
- Keeps local sidecar enforcement and local audit events inside the free/open-core boundary.
- Expands secret redaction coverage for common provider tokens and key material.
- Detects constructed or decoded egress destinations as review evidence.
- Adds local dependency extraction for additional lockfiles and local OSV advisory enrichment through `scan-skill` and `scan-skills`.
- Adds OpenClaw-native `doctor` checks for skill precedence, missing or wildcarded agent skill allowlists, and imported plugin diagnostics from `openclaw plugins`.

## Install

Install with Homebrew:

```bash
brew update
brew install runbrake/tap/runbrake
runbrake
```

Or use a checksummed binary release:

```bash
curl -fsSLO https://github.com/runbrake/runbrake-oss/releases/download/v0.1.1/runbrake_v0.1.1_darwin_arm64.tar.gz
curl -fsSLO https://github.com/runbrake/runbrake-oss/releases/download/v0.1.1/runbrake_v0.1.1_checksums.txt
grep " runbrake_v0.1.1_darwin_arm64.tar.gz$" runbrake_v0.1.1_checksums.txt | shasum -a 256 -c -
tar -xzf runbrake_v0.1.1_darwin_arm64.tar.gz
./runbrake doctor --path ~/.openclaw
```

## Verification

This release should be cut only after `pnpm run ci:check`, `scripts/build-release-artifacts.sh v0.1.1`, checksum verification, and public OSS CI pass.

## Limits

- Static findings and dependency advisories are review signals, not proof that a skill is actively malicious.
- Runtime observations are metadata-first and intentionally omit raw prompts, raw tool arguments, file contents, and unredacted secrets.
- Local sidecar policy decisions and audit events stay local unless the operator explicitly exports or shares them.
- `doctor --openclaw-bin` imports plugin diagnostics only when the operator provides an OpenClaw binary.
