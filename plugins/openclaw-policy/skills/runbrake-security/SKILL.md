# RunBrake Security For OpenClaw

## When To Use

Use this skill when the user asks for RunBrake status, recent RunBrake receipts, what RunBrake blocked, whether a new OpenClaw skill or plugin is safe, how to check new skills, how to turn enforcement on or off, or how to generate an assessment report.

This skill is meant to be usable from natural language and from slash-style invocation such as `/skill runbrake-security status`.

## Procedure

1. Classify the user's intent as status, recent receipts, block history, scan this skill, check new skills, enforcement guidance, sidecar setup, or report generation.
2. Keep the response privacy-safe. Do not quote raw tool arguments, prompt bodies, file contents, message bodies, memory contents, package bodies, or unredacted secrets.
3. If the current agent session has terminal access and the user asked for an action, run the matching command from `references/runbrake-commands.md` and summarize the result.
4. If terminal access is unavailable, explain the exact command and what a successful result means.
5. For RunBrake status, prefer sidecar health first, then local OpenClaw doctor output.
6. For recent RunBrake receipts, explain that Phase 3.8 receipts are emitted in-session by the plugin and can be correlated with local audit IDs; if a local receipt command is not available, use watcher and assessment commands to reconstruct recent skill/install risk.
7. For "what did RunBrake block?", report only receipt headlines, policy IDs, rule IDs, severity, and audit event IDs when available.
8. For "scan this skill", ask for the path if it was not provided, then use `runbrake assess --path <path>` or `runbrake scan-skills <path>` depending on whether the path is a single skill or an OpenClaw root.
9. For "check new skills", use `runbrake watch-openclaw --once --path ~/.openclaw --format console`.
10. For enforcement guidance, explain quiet/all/off receipt modes and the local sidecar policy file, but do not change policy unless the user explicitly asks.

## Response Shape

For status, use:

```text
RunBrake status: <active | fail-open | not configured>
OpenClaw path: <path>
Sidecar: <reachable | unavailable>
Receipts: <quiet | all | off>
Next check: <one command or next action>
```

For scans, use:

```text
RunBrake checked <target>: <clean | findings | command failed>
Highest severity: <info | low | medium | high | critical>
Top rule: <rule id or none>
Report: <path if generated>
```

## Notes

Runtime receipts are generated by the RunBrake OpenClaw policy plugin and local sidecar. This skill helps the user query and operate that local system from the agent UI; it does not upload receipts or create a hosted ledger.
